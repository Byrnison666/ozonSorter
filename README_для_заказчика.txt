================================================================
  OzonSorter — десктоп для сортировки посылок Ozon
  Сборка от 2026-05-16 (премиум-интерфейс)
================================================================

НАЗНАЧЕНИЕ
----------
Программа выбирает из ежедневного Excel-отчёта Казакова 68
только посылки наших клиентов, исключает уже лежащие у нас на
точке и формирует два Excel — на Комсомольскую 4 и Кольцевую 16.


СИСТЕМНЫЕ ТРЕБОВАНИЯ
--------------------
— Windows 10 или Windows 11 (64-bit)
— ~250 МБ свободного места на диске
— Права администратора НЕ требуются


УСТАНОВКА
---------
1. Распакуйте папку OzonSorter в постоянное место, например:
       C:\OzonSorter\
   ВАЖНО: не оставляйте её в Загрузках — папку могут случайно
   удалить, тогда программа перестанет запускаться.

2. Откройте папку и двойным щелчком запустите OzonSorter.exe

3. При первом запуске Windows может показать предупреждение
   «Защитник Windows предотвратил запуск неопознанного приложения».
   Это нормально для самописных программ без сертификата.
   Действия:
       — нажмите «Подробнее»
       — нажмите «Выполнить в любом случае»
   После этого Windows запомнит выбор и больше не спросит.

4. Если ваш антивирус (Касперский, Avast, Dr.Web) ругается на
   OzonSorter.exe — это ложное срабатывание (особенность сборок
   PyInstaller). Добавьте папку OzonSorter в исключения антивируса.


КУДА СОХРАНЯЮТСЯ ДАННЫЕ
-----------------------
База данных и логи лежат в папке пользователя:
       C:\Users\<ваше имя>\AppData\Roaming\OzonSorter\

Это значит:
— можно переносить папку с программой куда угодно — данные не теряются
— можно переустановить программу — клиенты и история останутся
— для бэкапа достаточно скопировать содержимое этой папки


ПЕРВЫЙ ЗАПУСК — ПОРЯДОК ДЕЙСТВИЙ
--------------------------------
1. Откройте раздел «Клиенты» в левом меню.
2. Нажмите «+ Добавить клиента», внесите 1–2 тестовых:
       — Ozon ID:    0224933356  (только цифры, как в номере отправления)
       — ФИО:        Иванов И.И.  (необязательно)
       — Телефон:                  (необязательно)
       — Политика:   FIXED          (если все посылки на одну точку)
                  или MANUAL         (если бывают на обе)
       — Точка:      Комсомольская 4 (только при FIXED)
3. Откройте раздел «Главная» и нажмите «Выбрать файл…».
4. Выберите Excel от Казакова 68 (формат «Остатки на складе ДОНЕЦК_26»).
5. Дождитесь сводки: сколько распознано, сколько новых к отгрузке.
6. Если есть MANUAL-клиенты — перейдите в «Распределение» и
   распределите их посылки по точкам.
7. На «Главной» нажмите по очереди:
       «Excel для Комсомольской 4»
       «Excel для Кольцевой 16»
   Каждый раз появится диалог «Сохранить как…» — выберите папку.
8. Передайте оба файла на Казакова 68.


ЕЖЕДНЕВНЫЙ ЦИКЛ
---------------
— Утром: получаете Excel от Казакова, импортируете, распределяете
  MANUAL-клиентов, формируете 2 файла, отдаёте на Казакова.
— В течение дня: после того как привезли посылки на точку — см.
  раздел «Остатки на точках». (На этой версии перевод посылок в
  статус «на точке» пока ручной через работу с БД — функционал
  «Подтвердить отгрузку получена» будет в следующей сборке.)
— Когда точки сообщают о выданных — раздел «Выдача»: вводите
  Ozon ID клиента в большое поле поиска, жмёте Enter, проверяете
  найденные посылки, нажимаете «Выдать выбранные».


ЧЕК-ЛИСТ ПРИЁМКИ (что проверить на ваших данных)
------------------------------------------------
[ ] 1. Программа запустилась, открылось окно «OzonSorter — Казакова 68»
       с тёмным сайдбаром слева и светлой контентной областью.
[ ] 2. Добавили клиента вручную в разделе «Клиенты» — он появился в
       таблице, рядом с политикой видна цветная плашка FIXED/MANUAL.
[ ] 3. Импортировали реальный Excel от Казакова — на «Главной»
       появились карточки с числами: «Всего строк», «Наших»,
       «Новых к отгрузке», «Уже на точках», «Не наши», «КТЯ».
[ ] 4. Сформировали два Excel — открыли в MS Excel, проверили:
       — в файле для Комсомольской только клиенты с привязкой к ней
       — в файле для Кольцевой — только её клиенты
       — повреждённые посылки выделены красным
       — три колонки: этикетка+название / номер отправления / ячейка
[ ] 5. Повторно импортировали тот же файл — программа предупредила
       (или просто не создала дубликатов).
[ ] 6. Если есть MANUAL-клиенты: проверили работу экрана
       «Распределение» — статусная плашка наверху корректно
       переключается между «X в очереди» и «Всё распределено».
[ ] 7. Поиск в разделе «Выдача» по реальному Ozon ID находит посылки.


ЧТО ДЕЛАТЬ ПРИ ПРОБЛЕМАХ
------------------------
1. Сначала закройте программу и запустите заново.
2. Если что-то выглядит сломанным — сохраните копию папки
       C:\Users\<имя>\AppData\Roaming\OzonSorter\
   и пришлите её разработчику вместе с описанием действий, после
   которых появилась проблема.
3. Логи лежат там же, в подпапке logs/
